If you have ever tried to install a third-party Android APK directly from the browser on a Chromebook or Chrome OS tablet, you have probably hit a wall: *"Turn on Chrome OS Developer Mode to install apps from sources other than the Play Store."* 

The official Google prompt tells you to enable Developer Mode—but doing so triggers a Powerwash, completely wiping your local data and disabling OS verification.

Fortunately, there is a much better, safer way to sideload APKs using the built-in Linux development environment. Here is how to bypass the factory reset and install any Android app using ADB.

### The Prerequisite: Enable Linux
Before starting, ensure your Chrome OS device supports the Linux environment.
1. Go to **Settings > Advanced > Developers**.
2. Turn on the **Linux development environment**.
3. Follow the on-screen prompts and leave the disk size at its default setting. The installation will take a few minutes.

### Step 1: Enable ADB Debugging
Once Linux is running, you need to create a bridge between the Linux container and the Chrome OS Android subsystem.
1. Navigate back to **Settings > Advanced > Developers**.
2. Click on **Linux development environment**.
3. Select **Develop Android apps**.
4. Toggle on **Enable ADB debugging**.
5. Your device will prompt you to restart. Accept it, and when the device boots back up, confirm the ADB debugging prompt on your screen.
