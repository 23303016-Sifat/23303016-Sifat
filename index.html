How to Sideload Android Apps on Chrome OS (Without Wiping Your Data)

If you have ever tried to install a third-party Android APK directly from the browser on a Chromebook or Chrome OS tablet, you have probably hit a wall: "Turn on Chrome OS Developer Mode to install apps from sources other than the Play Store."

The official Google prompt tells you to enable Developer Mode—but doing so triggers a Powerwash, completely wiping your local data and disabling OS verification.

Fortunately, there is a much better, safer way to sideload APKs using the built-in Linux development environment. Here is how to bypass the factory reset and install any Android app using ADB.

The Prerequisite: Enable Linux

Before starting, ensure your Chrome OS device supports the Linux environment.

  1. Go to Settings > Advanced > Developers.
  2. Turn on the Linux development environment.
  3. Follow the on-screen prompts and leave the disk size at its default setting. The installation will take a few minutes.

Step 1: Enable ADB Debugging

Once Linux is running, you need to create a bridge between the Linux container and the Chrome OS Android subsystem.

  1. Navigate back to Settings > Advanced > Developers.
  2. Click on Linux development environment.
  3. Select Develop Android apps.
  4. Toggle on Enable ADB debugging.
  5. Your device will prompt you to restart. Accept it, and when the device boots back up, confirm the ADB debugging prompt on your screen.

Step 2: Install ADB Tools in the Terminal

Open your app launcher and start the Terminal app. Click on the penguin icon to open the command line.

First, update your package list:

sudo apt update

Next, install the ADB (Android Debug Bridge) platform tools:

sudo apt install adb -y

Step 3: Prepare Your APK File

For security reasons, Chrome OS isolates your downloaded files from the Linux terminal. You have to manually move your APK into the Linux ecosystem.

  1. Open the Chrome OS Files app.
  2. Locate your downloaded APK in the Downloads folder.
  3. Rename the file to something simple with no spaces (e.g., appname.apk). Note: Linux is strictly case-sensitive, so keeping it all lowercase saves headaches later.
  4. Drag and drop the renamed APK into the Linux files folder on the left sidebar.

Step 4: Connect and Install

Head back to your terminal. Sometimes, the Linux container fails to automatically detect the internal Android emulator. We can force the connection using Chrome OS's standard internal IP for this bridge.

Run this command:

adb connect 100.115.92.2:5555

Immediately look at your screen. A pop-up will appear asking, "Allow USB debugging?" Check the box that says "Always allow from this computer" and click OK.

Now, push the install command:

adb install appname.apk

Once the terminal outputs Success, you are done! You can close the terminal, and your newly sideloaded app will be waiting for you in your standard Chrome OS app drawer.

Important Note: The Lock Screen Warning

After successfully setting this up, you will notice a permanent warning on your lock screen: "This device may contain apps that haven't been verified by Google."

This is completely normal. Because you opened an ADB loophole to bypass the Play Store, Chrome OS is just doing its job by notifying you that standard security is relaxed.

Do not turn off ADB debugging to get rid of the message! If you toggle ADB debugging off in your settings, Chrome OS will instantly factory-reset the Android subsystem for security, completely deleting any apps you just sideloaded. Leave it on, ignore the text, and enjoy your apps.
